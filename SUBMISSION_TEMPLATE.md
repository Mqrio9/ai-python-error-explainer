## One bug/issue handled with 5-step QA flow

Bug: Empty input caused a bad user experience.

1. Found issue: A user clicked "Explain it" without typing anything.
2. Reproduced: I opened the app, left the box empty, and clicked the button.
3. Cause: The app needed input validation before sending to AI.
4. Fix: I added a check that shows "Please paste a Python error or code first."
5. Retest: I tried again with an empty box, and the app showed the warning correctly.
